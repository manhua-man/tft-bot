## 2. 工具详解

### 2.1 终端 (terminal)
```
能力: 执行任意 shell 命令 (bash/git-bash on Windows)
场景: 构建、安装、git、网络、包管理、脚本执行
模式:
  ├── 前台执行 (默认) — 等待结果，适合短命令
  ├── 后台执行 — 返回 session_id，适合服务器/长任务
  └── PTY 模式 — 伪终端，适合交互式 CLI (Codex, Claude Code)
```

### 2.2 文件操作 (read_file / write_file / patch / search_files)
```
能力:
  read_file     — 读取文件，支持行号、分页、大文件
  write_file    — 写入文件，自动创建目录，语法检查
  patch         — 精确查找替换，9 种模糊匹配策略
  search_files  — 内容搜索 (ripgrep) 或文件名搜索 (glob)
```

### 2.3 浏览器 (browser_*)
```
能力: 完整的浏览器自动化
工具链:
  browser_navigate  — 打开 URL
  browser_click     — 点击元素
  browser_type      — 输入文本
  browser_snapshot  — 获取页面结构
  browser_vision    — 截图 + AI 分析
  browser_console   — 执行 JavaScript
  browser_scroll    — 滚动页面
场景: 表单填写、CAPTCHA、动态页面、Web 应用交互
```

### 2.4 图像 (vision_analyze / image_generate)
```
vision_analyze  — 加载图片并用 AI 分析内容
image_generate  — 文本生成图片 (支持 OpenAI/xAI/FAL 等后端)
```

### 2.5 子代理 (delegate_task)
```
能力: 将任务分配给独立子代理并行执行
模式:
  ├── 单任务 — 一个子代理完成一个目标
  ├── 批量 (最多 3 个并行) — 独立任务同时执行
  └── 编排者 — 子代理可以再派子代理
用途: 复杂推理、大型代码任务、并行研究
限制: 子代理无父对话上下文，需显式传递信息
```

### 2.6 定时任务 (cronjob)
```
能力: 创建定时/周期性任务
模式:
  ├── Agent 模式 (默认) — LLM 每次执行任务
  └── Script 模式 — 脚本直接执行，无 LLM
调度: cron 表达式 / 固定间隔 / 一次性
交付: 当前对话 / 指定频道 / 本地保存
```

### 2.7 代码执行 (execute_code)
```
能力: 执行 Python 脚本，可调用所有 Hermes 工具
限制: 5 分钟超时，50KB 输出，50 次工具调用
用途: 批量操作、数据处理、条件逻辑、循环
```

### 2.8 其他工具
```
memory       — 持久记忆 (跨会话保留事实)
todo         — 当前会话的任务列表管理
skills       — 加载技能知识
session      — 搜索历史对话
clarify      — 向用户提问确认
tts          — 文本转语音
```

  ✓ 安装软件包
  ✓ 后台长时间运行任务

需要用户确认的:
  ? 删除文件/目录 (会询问)
  ? 破坏性操作
  ? 发送到外部的服务

不能做的:
  ✗ 访问物理设备
  ✗ 跨机器操作
  ✗ 修改系统级配置
  ✗ 访问用户的浏览器会话/Cookie
